# 国税庁 アプリケーションID 発行申請 — 返信ガイド

> **5月7日(水) 期限** に向けて、国税庁 軽減税率・インボイス制度対応室への返信方法を確定させます。
> 根拠資料: 国税庁「適格請求書発行事業者公表システム Web-API 機能の利用手続について」第一編 1.5版 (令和7年2月)、第二編 1.5版 (令和6年5月)

## 状況

国税庁から下記のメールが届きました (2026-04-20):

> 以前、適格請求書発行事業者公表サイトのアプリケーションID発行届出フォームよりアプリケーションIDの発行届出を提出いただきましたが、発行を希望するアプリケーションＩＤに応じたメールの確認ができておりません。

2つの選択肢:

- **Option A**: インボイス Web-API 機能 + 法人番号 API の両方申請 (Excel 申請書 + 添付書類)
- **Option B**: 法人番号 Web-API のみ (メール3項目のみ)

## 戦略判断: **Option A を選択**

理由 (4点):

### 1. v0.3.0 ロードマップで必須

本 MCP プロジェクトの v0.3.0 (2026-07 予定) で **`lookup_invoice_issuer` tool** を追加し、T番号 (`T`+13桁) による適格請求書発行事業者の状態照会を実装する。Option B で法人番号のみ取得した場合、v0.3.0 時点で**別途インボイスAPIを再申請が必要**で、同一メールアドレスで複数ID取得不可 (第一編 別添1 第3条) のため、別メール作成・再申請・再承認の3重コストが発生する。

### 2. 法人番号API は「保険」として確保される

Excel 申請書の**項番4「法人番号システムWeb-API機能に係るアプリケーションID」** で「はい」を選択すれば、インボイスAPI の審査が通らなくても法人番号API の ID が発行される。

> 国税庁が Web-API 機能の利用を認めない場合に、発行申請書の４「法人番号システム Web-API 機能に係るアプリケーション ID」の「はい」にチェックがあったときは、法人番号システム Web-API 機能は利用ができますので、アプリケーションID のお知らせを送付します。 (第一編 §5.2)

つまり **Option A = v0.1.0 用の法人番号API を最低保証 + v0.3.0 用のインボイスAPI も同時申請**という最善手。

### 3. 審査を通過する正当な申請理由あり

本プロジェクトはインボイス制度 B2B KYC 用途で、OSS の公開 MCP サーバとして:

- プログラム概要を明示可能 (GitHub 公開)
- アクセス権限管理を実装 (`src/lib/mcp-logger.ts` で redact、`src/api/rate-limiter.ts` でレート制御)
- 個人情報保護設計済 (`docs/policy/privacy-policy.md`、`docs/security/prompt-injection-defense.md`)
- 監査対応可能 (Sugukuru 本社で対応)

これらを添付書類 (プログラム概要、本 MCP の GitHub リポジトリ URL) として提出可能。

### 4. 同じメールで両API が動く

第一編 別添1 第2条第5号で「アプリケーションID」 は「本機能 (インボイス) **や法人番号システム Web-API 機能の**利用者を特定するために提供する符号」 と定義されており、**単一ID で両API 使用可能**。再申請コストゼロで v0.3.0 に移行できる。

## Option A の提出手順

### 1. Excel 申請書のダウンロード

公表サイトから Excel ファイルを取得:

```
https://www.invoice-kohyo.nta.go.jp/web-api/index.html
```

または国税庁から受信したメールの指示に従う。

### 2. Excel の入力項目

別途 [`excel-application-fields.md`](excel-application-fields.md) に完全な入力案を記載。要旨:

- **1. 事業概要**: スグクル株式会社の情報
- **2. 取得した公表情報を利用するシステム**: `@drapt-lab/mcp-houjin-bangou` の概要
- **3. 誓約事項**: すべて「はい」
- **4. 法人番号システムWeb-API機能のみの利用希望の有無**: 「はい」(保険)

### 3. 添付書類 (PDF、各10MB以内)

2種類どちらかを提出。本プロジェクトでは **2.7.1 (プログラムの概要)** を推奨:

**2.7.1. プログラムの概要** (PDF化推奨):

- `README.md` の主要セクション (英+日併記)
- `docs/architecture.md` (Mermaid 図含む)
- GitHub リポジトリ URL: https://github.com/sugukurukabe/mcp-houjin-bangou
- 5 Tool + 3 Prompt + Resources の一覧 (`docs/api-reference.md` より)
- プライバシー方針 (`docs/policy/privacy-policy.md`)
- プロンプトインジェクション防御設計 (`docs/security/prompt-injection-defense.md`)

### 4. メール返信内容

国税庁への返信本文は [email-template-option-a.md](email-template-option-a.md) 参照。件名と宛先:

- **To**: `invoice-web-api@nta.go.jp`
- **Cc**: (なし、返信のみ)
- **Subject**: `Re: [国税庁からのお知らせ]アプリケーションID発行申請書のご提出 (スグクル株式会社)`
- **添付**: `houjin-bangou-mcp-program-overview.pdf` (プログラム概要) + `applciation-form-sugukuru.xlsx` (Excel 申請書)

スグクル株式会社の壁です。

件名の件、法人番号システム Web-API 機能の利用を希望いたします。
以下、ご指示の3項目を記載します。

(1) 利用者の氏名又は名称: スグクル株式会社
(2) アプリケーションID発行届出フォームで入力したメールアドレス: engineering@sugukuru.co.jp
(3) 法人番号システム Web-API の利用希望である旨: 法人番号システム Web-API 機能のみの利用を希望します。

お手数をおかけしますが、よろしくお願いいたします。

---
スグクル株式会社
壁
engineering@sugukuru.co.jp

**注意**: これを選ぶと v0.3.0 のインボイスAPI統合時に**再申請 + 別メール作成**が必要。時間がない場合の最終手段。
